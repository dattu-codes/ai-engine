User Guide

Welcome to the AI Engine Documentation. AI Engine is a commercial-grade SaaS platform enabling automatic repository analysis, code review scans, testing execution, and deployment readiness checks in unified workspace environments.

Getting Started

To analyze a repository, connect your GitHub account or upload a code repository ZIP file in the Projects tab. Once ingested, select the project and click Execute Workflow to start the code analysis pipeline.

Repository Insights

Each review calculates an aggregated repository maturity score out of 100 based on 6 core vectors: Architecture, Security, Testing, Deployment, Maintainability, and Documentation. You can view these dimensions and the compiled prioritized improvement roadmap on the Repository Insights page.

Architecture Overview

AI Engine runs using a modular, decoupled microservice queue pipeline designed for scalability:

[Client Dashboard] ---> [Nginx Reverse Proxy] ---> [FastAPI Gateway]
                                                           |
                                                           |---> [PostgreSQL Database]
                                                           |---> [Redis Queue Pool]
                                                           v
                                                [Background daemon Workers]
                                                           |
                                                           v
                                             [LLM / Ingestion Parser engines]

Infrastructure Layers

The backend routes write raw code payloads to an abstracted Storage Service (storing files either locally or in Amazon S3/Cloudflare R2 buckets). Jobs are enqueued into a thread-safe Redis-based FIFO list, where background daemon listener threads pop tasks and run checks sequentially.

Public API Authentication

The versioned REST API v1 is accessible at the /api/v1 route prefix.

To authenticate requests, generate an API token inside the Settings modal of the dashboard. Send the API key inside the X-API-KEY header of every request:

curl -X GET "http://localhost:8000/api/v1/projects" \
  -H "X-API-KEY: ae_your_api_key_hash_here"

REST API v1 Endpoints

GET /api/v1/projects

Retrieves all projects owned by the authenticated account.

POST /api/v1/projects?name=MyProject&repo_url=https://github.com/org/repo

Registers a new project workspace. Subject to plan gating limits.

POST /api/v1/projects/{id}/analyses

Triggers an asynchronous code review analysis for a project.

GET /api/v1/projects/{id}/findings

Returns the detailed list of bugs, issues, and maintainability concerns detected during scans.

GET /api/v1/projects/{id}/insights

Returns computed architecture scores, tech debt ratings, and engineering maturity levels.

Docker Compose Deployment

AI Engine can be deployed in minutes using the provided multi-stage Docker configurations.

Quick Start

Run the stack locally with:

docker-compose up --build -d

This starts Postgres (port 5432), Redis (port 6379), FastAPI Gateway (port 8000), Background Worker, and Nginx Reverse Proxy (port 80).
